## File Format Reference

Source spec: Westpac Corporate Online ‚Äî New Zealand Direct Entry file format (August 2011)

### Format rules

- Fixed-length records, exactly **180 characters** per record
- Records separated by `\r\n` (CRLF)
- Field padding: left-justified = blank-filled right; right-justified = zero-filled left
- Amounts in **cents** as integer (no decimal point), right-justified, zero-filled

### Header Record `A` ‚Äî 180 chars

| Pos   | Len | Field           | Rule                                      |
|-------|-----|-----------------|-------------------------------------------|
| 1     | 1   | Record type     | Hardcoded `A`                             |
| 2‚Äì7   | 6   | Sequence number | Right-justified, zero-filled. Always `000001` for single-header files |
| 8‚Äì9   | 2   | Origin Bank     | Hardcoded `03` (Westpac NZ)               |
| 10‚Äì13 | 4   | Origin Branch   | User input. Right-justified, zero-filled  |
| 14‚Äì43 | 30  | Customer name   | Optional. Left-justified, blank-filled    |
| 44‚Äì49 | 6   | Customer number | Not used. Must be blank-filled            |
| 50‚Äì69 | 20  | Description     | Left-justified, blank-filled. Appears as "Other Party Name" on bank statement |
| 70‚Äì75 | 6   | Date            | `DDMMYY`. Must be a valid date            |
| 76‚Äì180| 105 | Blank           | Blank-filled                              |

### Detail Record `D` ‚Äî 180 chars

| Pos     | Len | Field              | Rule                                                |
|---------|-----|--------------------|-----------------------------------------------------|
| 1       | 1   | Record type        | Hardcoded `D`                                       |
| 2‚Äì7     | 6   | Sequence number    | Right-justified, zero-filled. Auto-incremented from 2 |
| 8‚Äì9     | 2   | Payee Bank         | Valid NZ bank number. Right-justified, zero-filled  |
| 10‚Äì13   | 4   | Payee Branch       | Right-justified, zero-filled                        |
| 14‚Äì21   | 8   | Payee Account      | Right-justified, zero-filled                        |
| 22‚Äì25   | 4   | Payee Suffix       | Right-justified, zero-filled                        |
| 26‚Äì27   | 2   | Transaction code   | `52` = Payroll, `50` = General Payment              |
| 28‚Äì29   | 2   | MTS Source         | `DC` for Direct Credits (Phase 1 only)              |
| 30‚Äì44   | 15  | Amount             | Cents, no decimal. Right-justified, zero-filled     |
| 45‚Äì64   | 20  | Payee name         | Left-justified, blank-filled                        |
| 65‚Äì76   | 12  | Payee Particulars  | Appears on payee's statement. Left-justified, blank-filled |
| 77‚Äì88   | 12  | Payee Analysis code| Appears on payee's statement. Left-justified, blank-filled |
| 89‚Äì100  | 12  | Payee Reference    | Appears on payee's statement. Left-justified, blank-filled |
| 101‚Äì102 | 2   | Payer Bank         | Right-justified, zero-filled                        |
| 103‚Äì106 | 4   | Payer Branch       | Right-justified, zero-filled                        |
| 107‚Äì114 | 8   | Payer Account      | Right-justified, zero-filled                        |
| 115‚Äì118 | 4   | Payer Suffix       | Right-justified, zero-filled                        |
| 119‚Äì138 | 20  | Payer Name         | Left-justified, blank-filled                        |
| 139‚Äì180 | 42  | Spare              | Blank-filled                                        |

### Padding helpers

```javascript
const padL = (s, n) => String(s||'').substring(0,n).padEnd(n,' ');    // left-justify, blank-fill
```

### Amount handling
- UI accepts dollars with decimals e.g. `5491.13`
- Convert to cents: `Math.round(parseFloat(amount) * 100)`
- Display back as dollars: `(cents / 100).toFixed(2)`

### Date handling
- `<input type="date">` gives `YYYY-MM-DD`
- Convert to `DDMMYY` for A record: `dd + mm + yy.substring(2)`
- When parsing bank file date `DDMMYY` ‚Üí convert back to `YYYY-MM-DD` for the input:
  - Year: `parseInt(yy) < 50 ? '20' + yy : '19' + yy`

### Sequence numbers
- Header A record: always `000001`
- Detail D records: start at `000002`, increment per row

## CSV Template Format

### Downloadable template columns

```
payee_name, payee_bank, payee_branch, payee_account, payee_suffix,
amount, particulars, analysis_code, reference,
payer_bank*, payer_branch*, payer_account*, payer_suffix*, payer_name*
```
*optional ‚Äî overrides global payer defaults for that row only

### Example row (matches sample bank file)

```csv
payee_name,payee_bank,payee_branch,payee_account,payee_suffix,amount,particulars,analysis_code,reference,payer_bank,payer_branch,payer_account,payer_suffix,payer_name
Mr J B Hewson,12,3125,00074656,0000,5491.13,Resimac  Pay,,,,,,,
```

### CSV parsing rules
- First row must be header row with exact column names (lowercase, underscores)
- Required columns: `payee_name`, `payee_bank`, `payee_branch`, `payee_account`, `payee_suffix`, `amount`
- Optional columns: `particulars`, `analysis_code`, `reference`, and all `payer_*` fields
- Per-row payer override: only applied if at least `payer_bank`, `payer_branch`, `payer_account` are all non-empty
- Quoted CSV values supported
- XLSX: show message to save as CSV first (no external library dependency)

## Bank File (.txt) Import Rules

When loading an existing Westpac DE file:

1. **A record** ‚Üí pre-fill form fields:
   - Origin Branch ‚Üí Step 1 Branch field
   - Customer Name ‚Üí Step 1 Company Name field
   - Description ‚Üí Step 1 Description field
   - Date ‚Üí Step 1 Date field (convert `DDMMYY` ‚Üí `YYYY-MM-DD`)

2. **First D record** ‚Üí pre-fill default payer fields (Step 2):
   - Payer Bank, Branch, Account, Suffix, Name
   - Then clear those fields on the first transaction object so it uses defaults

3. **Subsequent D records** ‚Üí if payer fields match the defaults ‚Üí clear them (use default)

4. **Transaction code** detected from `D` record positions 26‚Äì27 ‚Üí set `currentTxCode` and update toggle UI

5. User can then: view transactions in table, delete individual rows, modify Step 1/2 fields, and re-download

## Output File

- Filename: `NZ_DE_DDMMYY.txt` (using the processing date)
- Encoding: UTF-8 (bank expects ASCII ‚Äî only use standard ASCII characters in inputs)
- Line endings: `\r\n` (CRLF) after every record including the last
- Each line: exactly 180 characters before the CRLF
